Cloud Applications Coding Test
AFL Fixture/Schedule Client
This is a challenge to test your abilities with React, and to see how you go about solving problems when
multiple approaches are available to you.
The Challenge
Your challenge is to use the Fixture API provided to build a client to view the AFL Fixture using React. If you
are unfamiliar with the AFL Fixture you can see an example here: AFL Fixture . There is a couple of mandatory
requirements, as well as a few “extra credits” bonus criteria, for if you are feeling adventurous.
Feel free to use component libraries like Shadcn and data fetching libraries like Tanstack Query .
Mandatory Requirements
• A list view of all matches from the fixture endpoint - https://0p0s8y841d.execute-api.ap-southeast-
2.amazonaws.com/test/react-code-test/fixture.
• Each match should have the following details displayed: Date, Team Names, Team Logos, Scores &
Venue.
• A filtered list view for each round in the fixture (i.e. show only Round 3 matches)
• You can use the following source for your logos for you application https://react-code-test.s3.ap-
southeast-2.amazonaws.com/logos/{code}.svg. Simply replace {code} with the team code from the
fixture API (i.e. Adelaide Crows = ADEL)
Outcomes
• Use of functional components
• Use of Typescript
• Good quality code organisation and folder structure
• Mobile-first design
• Good quality documentation in the form of README.md
Extra Credit: (In no particular order)
• Filter matches by venue
• Filter matches by team
